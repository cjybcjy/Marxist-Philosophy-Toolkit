# Concept Debugging

Use this file when the user asks what a concept means, challenges a concept, uses jargon loosely, or seems to know the name but not the operation.

## Core Rule

```text
Naming a concept is not understanding it.
```

A concept is understood only when it can explain a concrete relation, generate a testable distinction, and state its boundary.

## Debug Sequence

1. Name the concept.
2. State the real problem it was built to solve.
3. Translate it into one concrete example.
4. Give one counterexample or boundary.
5. Say what evidence would show the concept is being misused.
6. Return to the user's concrete problem.

## Concept Debug Card

```text
概念：
- [name]

它解决的现实问题：
- [what problem this concept was built for]

一句话操作定义：
- [what to look for in reality]

具体例子：
- [observable example]

反例/边界：
- [where the concept does not apply]

误用信号：
- [what would show this is slogan or jargon]

回到当前问题：
- [how it changes the user's analysis or action]
```

## Boundary Tests

| Concept | Must Clarify |
| --- | --- |
| Historical materialism | Which material production and social relation are being analyzed? |
| Productive forces | What capacity changed: tool, knowledge, labor organization, scale, or infrastructure? |
| Production relations | Who controls labor, capital, land, data, workplace, platform, or distribution? |
| Class | What position in production and surplus capture does the actor occupy? |
| State | What organized coercion, administration, legality, or public authority is involved? |
| Main contradiction | Which relation currently organizes the other tensions? |
| Practice | What action produces feedback that can revise the idea? |
| Mass line | Whose direct experience is collected, synthesized, returned, and tested? |
| Reform | What stage constraint is being adjusted, and what new contradiction may appear? |
| Sinicization | What method is preserved while concrete route changes? |

## Anti-Jargon Checks

- If the answer cannot give an example, it does not yet understand the concept.
- If the answer cannot give a boundary, it is using the concept as a slogan.
- If the answer cannot name what evidence would change the judgment, it is not practice-tested.
- If the answer uses a concept to end debate, it is probably ideology, not analysis.
- If the answer only quotes a term, load `references/source-map.md` and separate source from inference.
