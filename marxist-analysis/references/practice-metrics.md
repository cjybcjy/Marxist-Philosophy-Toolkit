# Practice Metrics

Use this file when an answer includes a practice test, experiment, learning loop, investigation, organization change, or personal action.

## Core Rule

```text
No metric, no practice test.
```

Practice is not "go try harder." It is action that produces evidence capable of revising the line.

## Metric Ladder

| Metric Type | Question | Examples |
| --- | --- | --- |
| Start metric | Did the practice actually begin? | opened file, contacted users, held meeting, shipped prototype |
| Process metric | Did the action expose reality? | interviews completed, direct observations, defect count, user replies |
| Result metric | Did the action change the situation? | conversion, retained habit, resolved blocker, reduced cycle time |
| Contradiction metric | Did evidence support or weaken the main-contradiction judgment? | bottleneck moved, actor behavior changed, cost source identified |
| Review metric | Did we revise the line? | continue, stop, shrink, expand, change contradiction judgment |

## Metric Design Card

```text
实践目的：
- [what judgment the action tests]

启动指标：
- [proof the action started]

过程指标：
- [proof the action contacted reality]

结果指标：
- [proof something changed]

矛盾判断指标：
- 如果看到 X，说明原判断更强。
- 如果看到 Y，说明原判断变弱。

复盘指标：
- 继续：
- 停止：
- 缩小：
- 扩大：
- 改判：
```

## Good Metrics

- Observable: someone can inspect whether it happened.
- Timed: define today, 7 days, 14 days, or one cycle.
- Small enough: failure does not destroy the next experiment.
- Tied to the contradiction: the metric changes the diagnosis, not just mood.
- Hard to game: avoid vanity metrics when possible.

## Bad Metrics

| Bad Metric | Why Bad | Better |
| --- | --- | --- |
| "I feel more confident" | Mood can change without reality contact | produced one artifact or got one external response |
| "I learned a lot" | Vague and non-reviewable | can explain concept with example and boundary |
| "Team alignment improved" | Easy to fake in meetings | decision cycle shortened or blocker owner named |
| "More views" | Vanity if not tied to goal | qualified replies, retention, conversion, or repeat behavior |
| "Worked harder" | Effort may hide wrong route | evidence gathered or bottleneck moved |

## Review Rule

At review time, do not ask only "did it work?" Ask:

```text
1. What did reality show?
2. Which assumption survived?
3. Which assumption broke?
4. Did the main contradiction move?
5. What is the next smaller or sharper test?
```
