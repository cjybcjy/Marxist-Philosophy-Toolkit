# Fallbacks

Use this file when the answer is drifting into slogans, roleplay, quote piles, overconfidence, or a plan that cannot be tested.

## Repair Tree

| Failure | Stop Doing | Repair |
| --- | --- | --- |
| Scenario is unclear | Do not start from theory categories | Load `references/scenario-checkpoints.md` and identify object, facts, and expected output |
| Facts are missing | Do not fill gaps with confident theory | Name the missing facts and give conditional conclusions |
| Cannot identify the main contradiction | Do not force a single label | List 2 to 3 candidate contradictions and define evidence that would choose between them |
| Answer becomes a slogan | Do not add more slogans or quotes | Return to conditions, actors, interests, stage, and practice test |
| Answer becomes a roundtable | Do not write "Marx says, Lenin says, Mao says" | Convert figure material into one theme-led analysis |
| User asks for action but answer stays abstract | Do not give a grand plan | Load `references/practice-cards.md` and produce one card |
| Route choice appears before constraints | Do not optimize or advise yet | Load `references/constraint-map.md` and identify the decisive constraint |
| Action has downside but no stop rule | Do not call it a practice test | Load `references/risk-and-inversion.md` and define failure path, reversibility, and stop condition |
| Concept is only named | Do not treat naming as understanding | Load `references/concept-debugging.md` and require example, boundary, and misuse signal |
| Practice has no metric | Do not call it practice检验 | Load `references/practice-metrics.md` and define start, process, result, contradiction, and review metrics |
| Current facts are unstable | Do not treat memory as evidence | Load `references/source-policy.md` and mark facts as verified or unverified |
| Repeated source anchors | Do not reuse the same few texts as universal proof | Load `references/source-coverage.md` and decide whether to broaden source clusters, admit a source gap, or route through a theme |
| Theory modules conflict | Do not smooth over tension | Load `references/arbitration.md` and compare conditions |
| User asks for source | Do not give a quote pile | Load `references/source-map.md` and separate source, context, inference, and boundary |

## Default Recovery Sequence

When stuck, restart the analysis in this order:

```text
1. What concrete object is being analyzed?
2. What facts are known, missing, or current-fact sensitive?
3. Which theme owns the problem?
4. What are the candidate contradictions?
5. Who controls resources, labor, legitimacy, data, money, or organization?
6. What stage permits and blocks action?
7. What one small test could change the judgment?
```

## Conditional Judgment Pattern

Use this when evidence is incomplete:

```text
现在只能作方法判断，事实判断还缺 X。

如果 X 为真，主要矛盾更可能是 A。
如果 X 不成立，主要矛盾应改判为 B。

下一步先验证：
- 证据：
- 方法：
- 复盘时间：
```

## Anti-Shortcut Rules

- Do not use "符合国情" to skip class, production-relation, contradiction, or practice analysis.
- Do not use "发展生产力" to skip distribution, dependency, ownership, or organization questions.
- Do not use "群众路线" to mean polling or pleasing everyone.
- Do not use "实践检验" as mere results worship; define what result proves, weakens, or revises the line.
- Do not use "主要矛盾" as a permanent label; state the stage and revision condition.
- Do not use the same famous article as evidence for every unrelated problem; check source coverage first.
- Do not turn personal problems into moral defects before mapping conditions and small tests.
