# Method Templates

Use this file when a question needs a repeatable analysis procedure rather than only a concept definition or source anchor.

## Core Rule

```text
Templates standardize the reasoning steps; they do not decide the answer.
```

Always fill templates with the user's concrete object, known facts, missing facts, source boundary, and practice test. If a template cannot be filled, use `fallbacks.md` instead of forcing a conclusion.

## Template Selection

| User need | Template |
| --- | --- |
| Priority, conflict, strategic focus | Contradiction analysis |
| "Is this true?" or "What should I test?" | Practice verification |
| Industry, policy, organization, social change | Historical-materialist analysis |
| Trend, stage, transformation, renewal | Development trend / negation analysis |

## Contradiction Analysis Template

Use for main contradiction, route choice, organizational conflict, or strategy.

```text
1. Define the object and stage.
2. List visible tensions without deciding yet.
3. Separate symptoms from structural contradictions.
4. Identify candidate main contradictions.
5. For each candidate, ask: if solved, what else changes?
6. Name the main contradiction and its principal aspect only if evidence supports it.
7. Identify transformation conditions.
8. Choose the smallest action that can test or shift the contradiction.
9. Set a review point and revision rule.
```

Minimum output:

```text
主要矛盾候选：
- A:
- B:

暂判：
- 主要矛盾：
- 主要方面：
- 转化条件：

实践检验：
- 小测试：
- 证据：
- 修正条件：
```

Boundary:

- If there is not enough evidence, list candidates and evidence needed.
- Do not turn every conflict into antagonistic struggle.
- Do not reuse `On Contradiction` as a universal answer without concrete object and stage.

## Practice Verification Template

Use for investigation, learning, product experiments, route validation, or personal practice.

```text
1. State the hypothesis or line being tested.
2. Define the practice field: where action can produce feedback.
3. Define start metrics: current state.
4. Define process metrics: actions actually taken.
5. Define result metrics: observable change.
6. Define contradiction metrics: what would reveal the hidden constraint.
7. Define stop, continue, or revise conditions.
8. Run a short review loop.
```

Minimum output:

```text
假设：
实践场：
启动指标：
过程指标：
结果指标：
矛盾指标：
复盘时间：
修正规则：
```

Boundary:

- Practice is not "whatever works once"; it must specify what result proves or weakens.
- A short-term result does not settle distribution, dependency, or long-term capacity.

## Historical-Materialist Analysis Template

Use for economy, industry, technology, policy, class, state, organization, or social reproduction.

```text
1. Define the material process: what is produced or reproduced?
2. Identify productive forces: tools, labor skill, science, organization, infrastructure, data.
3. Identify production relations: ownership, control, coordination, distribution, dependency.
4. Identify economic base and institutional/ideological forms that stabilize it.
5. Map classes or actor positions by relation to production and surplus capture.
6. Identify what relation now releases or fetters capacity.
7. Identify who gains, who pays, who bears risk, and who controls the bottleneck.
8. Name the stage and the possible transition.
9. Define current facts needed before strong judgment.
```

Minimum output:

```text
物质过程：
生产力：
生产关系：
利益/阶级位置：
上层建筑或组织形式：
卡点：
阶段判断：
事实缺口：
```

Boundary:

- Do not reduce historical materialism to money or GDP.
- Do not treat technology as automatically progressive.
- For modern industries, use current evidence rather than classic texts as fact substitutes.

## Development Trend / Negation Template

Use when the user asks where a thing is going, whether a route is exhausted, or how old and new forms transform.

```text
1. Define the old form and what problem it once solved.
2. Identify the new contradiction produced by the old form's success or limit.
3. Identify quantitative accumulation: repeated small changes.
4. Identify possible qualitative turning point.
5. Identify what is preserved, negated, and reorganized.
6. Separate necessary trend from contingent path.
7. Name evidence that would confirm or weaken the trend.
8. Translate into a staged route or watchlist.
```

Minimum output:

```text
旧形式：
旧形式解决过什么：
新矛盾：
量变信号：
质变门槛：
保留/否定/重组：
趋势证据：
反证：
```

Boundary:

- Do not use "否定之否定" to claim progress is guaranteed.
- Do not confuse an appealing narrative with a necessary trend.
- If facts are missing, give a watchlist rather than a prediction.

## Template Integrity Check

Before final answer, check:

- Did the template start from an object, not a slogan?
- Did it distinguish method, source, current facts, and inference?
- Did it include a boundary or failure condition?
- Did it produce a next investigation or practice step?
- Did it avoid turning textbook order into a textbook answer?
